AgileSprint App

Mobile | Responsive | Prototype

I designed an that hooks up to Trello’s API and allows for each member of the team to check in their progress as well as to see the advances of the entire team in the current sprint.


The Project

As the team grows, it’s harder to keep track of tasks let alone of each sprint. Whiteboards and Post-its were not cutting it. They had been using Trello for PM but it’s cumbersome to update and doesn’t represent progress in a visual way.

As I was doing the research, I came accross Trello’s API which inspired me to build the Agile App’s functionality on top of it as an extension.


The Goals

  • Low learning curve and simple
  • Completely responsive
  • Avoid function overlapping with current PM tools
  • Fun and playful UI

1 Week, 1 Sprint and 1 Designer

After an full day of research and another of wireframing, I jumped into Sketch and started to define the main components of the UI. I spent Thursday prototyping. It had to be ready for Friday.




How it Works


The app has 2 main views: Check In and Dashboard.
In Check In, you can update the status on each task that’s assigned to you. If you choose ‘Blocked’ you can send a message to the team. The message is posted as comment to the card via Trello’s API.

Desktop Version


The widgets on the Dashboard section use the color-coded iconography of the four possible status to mark their variation throughout the week.
To make better use of the space, the desktop version displays both views at the same time.



Mobile Version

The mobile prototype was interactive just like a native app which made it easy to test among the rest of the team. They loved it!
